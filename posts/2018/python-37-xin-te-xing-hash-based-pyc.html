Python 3.7 新特性：Hash-based pyc

星期一, 六月 04, 2018 on

熟悉 Python 的朋友都知道，Python 代码文件在第一次执行之后都会生成一个以 .pyc 为后缀储存 Python 执行的字节码的缓存文件，作用是在源码文件没任何修改并被再次执行时不用重新编译，提高程序启动的效率。

Python 在生成 pyc 文件时会把对应的源码文件的文件大小和最近修改时间作为元数据存放在 pyc 文件里。这种方式实现简单而且速度快，但是存在着一个问题：当被修改后的源码文件的文件大小没有变化时，某些操作系统的时间戳粒度太大，导致 Python 的编译器无法检测到修改时间的变化而误认为源码文件没被修改，使得仍使用旧的 pyc 运行程序。

这个虽然不常见，但遇到了却很难被发现。

因此 Python 3.7 为了解决这个问题引入了新特性—— Hash-based pyc。原理很好理解，与使用源码文件的大小和修改时间作为元数据的方式不同，hash-based pyc 文件保存了源码文件的 hash 值，在每次编译时会使用 pyc 文件中保存的 hash 值对当前源码文件 hash 值作比较，这。

Python 3.7 支持两种模式的 hash-based：checked 和 unchecked。checked模式就是上述的检测方式，而 unchecked 即简单地认为已有的 pyc 文件是合法的。

hash-based 的优点是比基于时间戳的检测方式更准确地检测到源码文件变化，缺点是需要对每个源码文件进行 Hash 操作，使得程序启动的速度变慢。
