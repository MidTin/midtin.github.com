Ubuntu Python环境+Vim配置 注意事项

星期三, 九月 03, 2014 on Python, Vim, Ubuntu

以前曾经尝试使用CentOS，不过感觉YUM没APT好用，于是选择了Ubuntu作为重新学习Linux的操作系统，毕竟Ubuntu在内地的受支持度还是很高的，至少没被qiang→_→

虚拟机：VMware Workstation 9

系统：Ubuntu 14.04.1

使用 VMware-Workstation 安装Ubuntu，当如CPU是64位，而安装系统是32位，则会在开机时提示

piix4_smbus 0000:00:007.3: Host SMBus controller not enabled

这是因为 Ubuntu 找不到 i2c_piix4 模块所致，

解决方法：在/etc/modprobe.d/blacklist.conf 末尾加入 blacklist i2c-pii4x

Python

在Ubuntu 14.04中已安装了 Python2.7.6 和 Python3.4.0，在终端输入python进入Python交互模式默认版本是 Python2.7.6，当需要使用 Python3.x 时就会很麻烦。

删除/usr/bin/python，然后重新建立软链接

# ln -s /usr/bin/python3.4 /usr/bin/python

这样就可以解决问题了。

可是重启系统后提示系统错误，/usr/lib/cups/backend/hpfax发生错误，

原因是/usr/lib/cups/backend/hpfax使用的是 Python2.x，

我们只需编辑它，把文件头#!/usr/bin/python改为#！/usr/bin/python2即可。

更新：上述方法过于 dirty，其实使用 pyenv 管理 Python 版本更加方便

Vim

一般我们选择安装vim-gnome

# apt-get install vim-gnome

这是一个有Gui的Vim，优势在于可使用标签分页。

如果是想让终端的Vim支持鼠标操作，在~/.vimrc中加入set mouse=a即可。

在切换到root用户时，如果使用su切换，第一次打开gvim是会提示错误

……
(gvim:2515): GConf-WARNING **: Client failed to connect to the D-BUS daemon
……

解决办法是使用su -切换到root用户。
