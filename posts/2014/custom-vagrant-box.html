如何自定义一个使用ssh证书验证登录的VBox?

星期五, 十一月 21, 2014 on Vagrant, Ubuntu

环境：Ubuntu-14.04.1-server-i386，Virtualbox 4.3.16，Vagrant 1.6.5，Win8.1*

如何创建自定义 VBox 可参考官方文档，本文仅叙述某些要点及笔者在过程中遇到的某些难题。

另外，这篇官方文章也是很有用处，里面提及了一些关于创建自定义开放版VBox时需要注意的地方。

配置 SSH

关于如何配置 SSH 证书登录验证的网上教程有很多，简单概括为以下几步：

  1. 使用 ssh-keygen 生成 private/public key 文件：

    ssh-key -t rsa -C 'user@email.com'

  2. 把~/.ssh/id_rsa移动到其它地方，以后就是使用这个key文件进行验证登录。

  3. 把~/.ssh/id_rsa.pub改名为authorized_keys，修改一下文件权限作为保护:

    sudo chmod 400 ~/.ssh/authorized_keys

  4. 编辑/etc/ssh/sshd_config， 把AuthorizedKeysFile %h/.ssh/authorized_keys一行注释去掉，在下面加上：

    PasswordAuthentication no 
PermitEmptyPasswords no

重启ssh服务后就完成整个配置。

Vagrant

创建完 VBox 后，我们通过它建立自己的 Vagrant 虚拟机，

在主机的你希望建立 Vagrant 虚拟机的目录，Shell 中输入：

vagrant box add boxsname your_boxs_path\yourbox.box
vagrant init boxsname
vagrant up
..

注意，由于我们使用的是 SSH 证书登录验证，我们vagrant up之前需要在刚生成的Vagrantfile中加入以下设置：

# vagrant把VirtualBox的端口转发规则改为主机端口2222->虚拟机端口22，所以这里填2222
config.ssh.port = 2222

# 配置ssh登录使用的用户名
config.ssh.username = 'linuxs_user_name'

# 配置private_key文件
config.private_key_path = 'your_private_key_path'

Vagrant up

在vagrant up生成虚拟机的过程中我们有可能遇到以下几个错误提示：

  1. default: Warning: Connection refused. Retrying...

    原因：一般是没配置好 SSH 导致验证登录失败

    解决办法：根据上面配置Vagrantfile可解决

  2. sudo: no tty present and no askpass program specified

    原因：Vagrant 在配置虚拟机时需要使用sudo，而一般的用户使用sudo需要输入密码，导致此错误的发生

    解决办法：修改/etc/sudoers的配置

    # 切换到root用户
su -

# 为/etc/sudoers增加写入权限
chmod 600 /etc/sudoers

# 编辑/etc/sudoers
vim /etc/sudoers

------------------------------------------------------------

:::linenums=True
# 在 /etc/sudoers中加入以下几行
yourusername        ALL= (ALL) NOPASSWD:ALL
Defaults:yourusername   !requiretty

#修改%sudo...
%sudo       ALL=(ALL) NOPASSWD:ALL

  3. /sbin/mount.vboxsf: mounting failed with the error: No such device

    原因：在创建VBox时Linux中需要安装guest-additions，若在安装完后Linux-kernel进行了更新，则会导致此错误的发生。据观测，即使更新了Vagrant虚拟机的guest-addtions，在下一次启动时也有可能出现这个问题。

    解决办法：安装第三方插件vagrant-vbguest，该插件会自动探查guest-additions的版本是否合适，若不适合则会自动进行guest-additions更新，十分方便。
