CentOS 升级 Git

Thursday, May 15, 2014 on Linux, Centos

现在Git的版本已经是1.7.12了，但CentOS-6.5的Git版本却是1.7.1，而用yum安装的Git的最高版本也只有1.7.1。那么我们该如何升级我们的Git呢？以CentOS-6.5为例：

安装RPMForge源

RPMForge源是什么？

RPMForge是CentOS系统下的软件仓库，拥有4000多种的软件包，被CentOS社区认为是最安全也是最稳定的一个软件仓库。

RPMForge地址：http://rpmfusion.org

首先，使用rpm远程安装需要的证书

# rpm --import http://apt.sw.be/RPM-GPG-KEY.dag.txt

之后，到RPMForge的http://pkgs.repoforge.org/rpmforge-release/选择适合自己系统的rpm安装包

# rpm -i http://pkgs.repoforge.org/rpmforge-release/rpmforge-release-0.5.3-1.el6.rf.i686.rpm

更新rpmforge-extra源

# yum --enablerepo=rpmforge-extras update

升级Git

查看yum库可用的git软件包

# yum --enablerepo=rpmforge-extras provides git

选择适合的Git版本安装

# yum --enablerepo=rpmforge-extras install gitk-1.7.12.4-1.el6.rfx.i686.rpm

检查Git版本

# git --version
